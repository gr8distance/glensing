# CL Package Registry — v0.5.0 Visual UI 要件定義

## 前提

- プロジェクト: area51の対になるパッケージレジストリ（hex.pm / RubyGems.org / crates.io 相当）
- 名前: 未定（候補: gargantua / iscandar）
- TLD: 名前とセットで決定（.space / .dev / .io）
- インフラ: Cloudflare Pages（静的サイト）
- v0.1.0: パッケージ一覧 + JSON検索（シンプルHTML）が前提として完成済み

---

## 設計原則

**ビジュアルはドキュメントである。**

テキストで説明するのではなく、UIそのものがレジストリの概念・area51との関係・エコシステムの状態を直感的に伝える。サイトを開いた瞬間に「これは何か」がわかること。

参考: Kaspersky Cybermap（https://cybermap.kaspersky.com/ja）の宇宙版。サイバー攻撃が地球上を飛び交うビジュアルを、パッケージが宇宙からarea51に飛来するビジュアルに変換する。

---

## 世界観

```
ユーザーのローカルマシン（地球）
  └── area51（パッケージマネージャー、~/.area51/packages/）
        ↑ fetch / install
レジストリ（宇宙空間）
  └── 無数のパッケージ（星・光点）
```

- area51 = 地上の秘密基地。パッケージを受け入れる港
- レジストリ = 宇宙空間。パッケージが存在する広大な場所
- パッケージ = 星・光点。area51に向かって飛来する

名前がgargantuaの場合: ブラックホールがパッケージを引き寄せる重力源
名前がiscandarの場合: 遠い惑星。area51から取りに行く目的地

---

## テキストなしで伝えるべきメッセージ（4つ）

1. レジストリは広大な宇宙空間にある — パッケージの集積地であり、無限の可能性を内包する場所
2. パッケージは星のように無数に存在する — 一つ一つが光を持ち、探索可能
3. `area51 install` するとパッケージが飛来する — 宇宙からローカルマシンへの旅路
4. エコシステムは生きていて、成長している — 静的なリストではなく、動的な宇宙

---

## 機能要件

### メインビジュアル（ヒーローセクション）

- 暗い宇宙空間を背景に、中心にブラックホール or 惑星を配置
- 降着円盤 or 大気圏のビジュアル表現（パッケージ集積地のメタファー）
- 無数のパッケージが光の軌跡（光線・パーティクル）として飛来するアニメーション
- 各パッケージは星のように発光し、軌跡を残す
- ユーザーのローカルマシン（地球）上のarea51がパッケージを受け取るイメージの表現

### インタラクション

- パッケージ（光点）をホバーすると名前・説明がツールチップで表示
- パッケージをクリックすると詳細ページに遷移
- リアルタイム感のあるアニメーション（実際のinstall数に連動する必要はないが、生きている感じを出す）
- パッケージ検索UIはビジュアル上にオーバーレイ or スクロール後に表示

### パッケージ検索（v0.1.0からの継続）

- v0.1.0で実装済みのクライアントサイドJSON検索をビジュアルUIと統合
- 検索結果がビジュアル上でハイライトされる等の連動があると望ましい（must-haveではない）

---

## 非機能要件

### パフォーマンス

- 初回ロード: 3秒以内にメインビジュアルが表示されること
- アニメーション: 60fps維持（モバイルでは30fps許容）
- 重いWebGL処理はプログレッシブに読み込む（スケルトン → ローポリ → フル描画）

### モバイル対応

- Three.js/WebGLが重い場合のフォールバック用静的ビジュアルを用意
- タッチ操作でのパッケージ選択に対応
- ビューポートに応じたレスポンシブ配置

### 初期パッケージ数への対応

- パッケージが少ない段階（10〜50個）でも寂しく見えない工夫
  - 銀河の星々として背景に装飾的なパーティクルを散りばめる
  - 登録済みパッケージは他の星より明るく・大きく表示
  - パッケージ数が増えるほど自然にビジュアルが壮観になる設計

### ダークテーマ

- 宇宙空間を表現するためダークテーマ基調は必須
- ライトテーマ対応は不要

---

## 技術スタック候補

- Three.js / WebGL: メインビジュアル（ブラックホール/惑星、降着円盤、宇宙空間）
- パーティクルシステム: パッケージの飛来表現
- GLSL シェーダー: 重力レンズ効果（簡易版）、降着円盤の発光
- Cloudflare Pages: ホスティング（静的サイトのまま）
- バニラJS or 軽量フレームワーク: 検索UI（v0.1.0からの継続）

---

## area51 CLI側の連動（任意）

- `area51 install` 実行時のメッセージに世界観を反映
  - 例: `Fetching from gargantua...` / `Resolving via iscandar...`
- レジストリのURLをarea51のconfigに設定可能にする

## 命名候補（TLDとセットで決定）

### gargantua

- 出典: Interstellar (2014)
- 意味: ブラックホール。あらゆるものを引き寄せる巨大な重力源
- UIとの親和性: ブラックホール + 降着円盤のビジュアルが名前の意味を直接説明する。ビジュアルがドキュメントであるという原則の最も強い体現
- TLD候補: gargantua.space（推奨） / gargantua.dev / gargantua.io

### iscandar

- 出典: 宇宙戦艦ヤマト (1974)
- 意味: ヤマトが目指した遠い惑星。必要なものを取りに行く目的地
- UIとの親和性: 惑星ビジュアル。area51から遠い惑星にパッケージを取りに行く構図
- イースターエッグ: イスカンダル（Iskandar）= アレクサンドロス大王のペルシャ語読み。CLの最重要ライブラリ `alexandria` はアレクサンドロスが建設した都市に由来。area51から最初にinstallするパッケージがalexandriaであり、その名の由来がイスカンダルと同一人物。表のレイヤー（SF）と裏のレイヤー（CL文化）の二層構造
- TLD候補: iscandar.space（推奨） / iscandar.dev / iscandar.io

## スコープ外（v0.5.0では対応しない）

- ユーザー認証・ログイン
- パッケージのpublish機能（area51 publishはarea51側の機能）
- ダウンロード統計のリアルタイム連動
- ライトテーマ
